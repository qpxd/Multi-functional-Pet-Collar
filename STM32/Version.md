# 喜戈玛毕设模板发布记录
#### 当前版本：V1.0.8
#### 发布日期：2023年02月17日
#### 模板路径：192.168.2.9\xgm\005_毕设模板\V1\template_v1.0.8.zip
#### 更新内容：见根目录下 Version.md 文件

V1.0.0版本： 
1. 初始版本发布（基于原demo）

V1.0.1版本:  
1. 修改 IIC 驱动，修改OLED驱动，从第0列开始显示，上电后延时初始化，修改MDK配置，SRAM大小等
2. 重构工程框架，驱动，外设，组件模块化， 修改头文件包含方式，驱动分层，明确依赖关系
3. 删除sys_boot组件以及一些资料文件
4. ds3231 时间结构体改为 u8 , TFT 注释不用的字符编码
5. 添加自动生成工具 （V1.0.0）

v1.0.2版本:	
1. 新增 MPU6050, HLK-V40, DS1302驱动
2. 修改上位机没有拷贝字库头文件的BUG
3. 修改串口底层驱动，添加模式选择参数，可以使用其中断模式
4. MPU6050 添加DMP库

V1.0.3版本:    
1. onchip模块中添加 flash 驱动， nvs实现
2. 重写servo舵机驱动，调整pwm模块部分引脚定义
3. 修改mpu6050未初始化获取加速度接口的bug
4. 修改任务优先级的配置，修改部分任务默认栈大小
5. hc_sr04模块中的定时器初始化合并到 create 函数中
6. 添加AT命令模块
7. 添加 VL53L0X 激光测距模块驱动
8. 修改工程配置 默认添加 switch 模块 

V1.0.4版本:
1. 修改AT命令模块部分参数
2. 所有使用串口的驱动模块 修改初始化结构 默认使用DMA串口模式
3. 修改NVS中的一些BUG
4. sys_cfg 优先级配置相关回退到之前版本不然esp8266可能初始化失败

V1.0.6版本:
1. 修改NVS中的一些BUG
2. 回退IIC驱动，和最初版本保持一致，解决max30102通信错误的问题，但是不能和其他设备共用总线

V1.0.7版本:
1. 修改SPI中的部分BUG, 使其SPI总线上能挂载多个SPI设备 
2. 移植且修改SD卡驱动, 解决不能喝TFT屏幕共用的问题 
3. 移植FATFS文件系统 
4. 修改Freertos默认堆大小和FLash大小
5. SIM800 添加获取短信数量，读取短信内容和删除短信的接口
6. esp8266 修改数据发送长度为0时 申请内存失败的bug

V1.0.8版本:
1. 添加矩阵键盘驱动测试完成
2. 新增AS608驱动 
3. TFT驱动添加中文显示功能
4. 修改AT指令模块 
5. HX711移到头文件中可以修改
6. esp8266 ONENET实现多组数据发送和字符串类型数据发送
V1.0.9版本:
1. 添加EC600N 上传阿里云服务器的驱动代码

V1.0.10版本:
1. HelpLayer 组件添加宏定义，使能/失能 各个模块
2. algorithm.h 文件中的 static 变量定义放在c文件里面，减少使用内存
